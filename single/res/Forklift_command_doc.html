APP → 设备 指令接口文档（铲车）

本文描述：APP 如何发送“行走 / 转向 / 大臂 / 铲斗 / 灯光 / OTA”等指令给指定铲车设备（通过服务端转发）。
原则：一次只发送 一条 指令。
快捷：Ctrl + K 搜索本页 ｜ 点击左侧目录快速跳转

1. 说明

每次发送一条指令，服务端按设备序列号（SN）转发给对应设备执行。
铲车停止动作建议按固件现状使用：+ol（停行走/铲斗相关）与 +or（停转向/大臂相关）。

2. 指令格式

发送指令统一格式如下（一次只发一条）：
+F+SN+指令-

2.1 模板

+F+{SN}+{CMD}-

2.2 字段说明

字段说明
SN设备序列号/识别码（用于定位目标设备）。
CMD本页“指令列表”中的任意一条指令（例如 +xs#1）。
-结束符（必须带）。

3. 指令列表

指令以 + 开头。
有值：+TYPE#VALUE；无值：+TYPE。
TYPE 指令写法（不带末尾 -） 说明（按固件执行逻辑） 取值
xs +xs#V 行走前进/后退：V=1 前进；V=2 后退 1 / 2
ws +ws#V 转向：V=1 左转；V=2 右转（会触发闪烁逻辑） 1 / 2
dx +dx#V 大臂：V=1 升；V=2 降 1 / 2
lz +lz#V 铲斗：V=1 上翻；V=2 下翻 1 / 2
ol +ol 停止行走/铲斗相关输出（固件将一组 GPIO 拉低） —
or +or 停止转向/大臂相关输出，并停止闪烁 —
HO +HO#V 左右灯常亮开关（LED 可能反相）：2000 开 / 1000 关 2000 / 1000
HP +HP#V 双闪：2000 开 / 1000 关 2000 / 1000
UP +UP 触发 OTA 检测 —

4. 示例（可复制）

示例 1：前进
行走前进（V=1）
+F+{SN}+xs#1-
示例 2：后退
行走后退（V=2）
+F+{SN}+xs#2-
示例 3：左转
转向左（V=1）
+F+{SN}+ws#1-
示例 4：升大臂
大臂上升（V=1）
+F+{SN}+dx#1-
示例 5：铲斗上翻
铲斗上翻（V=1）
+F+{SN}+lz#1-
示例 6：停止行走/铲斗相关
松手建议发送
+F+{SN}+ol-
示例 7：停止转向/大臂相关
松手建议发送
+F+{SN}+or-
示例 8：触发 OTA 检测
升级检查
+F+{SN}+UP-

5. 注意事项

1) 每条指令必须以 - 结尾。
2) APP目前未定义 V=0 的停止语义，停止主要依赖：+ol / +or。
3) 建议“松手即停”：松行走/铲斗 → 发 +F+SN+ol-；松转向/大臂 → 发 +F+SN+or-。
